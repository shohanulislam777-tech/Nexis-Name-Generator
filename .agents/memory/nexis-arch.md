---
name: Nexis architecture decisions
description: Non-obvious decisions for the Nexis licensing platform that future sessions must stay consistent with.
---

**Hand-rolled JWT:** No `jsonwebtoken` package. Uses HMAC-SHA256 via Node `crypto.createHmac`. Secret from `SESSION_SECRET` env. 7-day expiry. Implemented in `artifacts/api-server/src/lib/auth.ts`.

**Why:** Keep the bundle lean; avoid version drift on a security-critical dep.

**How to apply:** If adding token refresh or role-based scopes, extend the same `createAdminToken`/`verifyToken` pair — do not introduce a new JWT library.

---

**Users are denormalized:** No standalone users table. "Users" are derived at query time by aggregating `licenses.user_email` / `user_name`. The `/admin/users` route groups licenses by email.

**Why:** Licenses are the core entity; user identity is optional metadata on a license.

**How to apply:** If you need persistent user accounts (e.g. for a self-service portal), add a `users` table and FK from `licenses.user_id` — but don't assume one exists now.

---

**License key format:** `NEXIS-XXXXXX-XXXXXX-XXXXXX-XXXXXX` (4 × 6 hex chars uppercase). Generated by `generateLicenseKey()` in `lib/auth.ts` using `crypto.randomBytes(3).toString('hex').toUpperCase()`.

---

**Chrome Extension NEXIS_API_BASE:** Must be updated in three files before loading the extension: `extension/nexis/license.js`, `extension/nexis/content.js`, `extension/nexis/sidepanel.js`. Points to the deployed backend `/api` base URL.

**Extension icons:** `extension/nexis/assets/` requires real PNG files (icon16/32/48/128.png) — Chrome refuses to load unpacked extensions without them.

---

**Password hashing:** HMAC-SHA256 with hardcoded salt `"nexis-salt-v1"`. Default admin: username `admin`, password `admin123`. Must be changed in production.

**Why:** Deterministic, no bcrypt dep. Acceptable for internal admin panel; not recommended for user-facing auth.

---

**API codegen:** Source of truth is `lib/api-spec/openapi.yaml`. After any spec change, run `pnpm --filter @workspace/api-spec run codegen` to regenerate hooks in `lib/api-client-react/src/generated/api.ts` and Zod schemas in `lib/api-zod/src/generated/api.ts`.
